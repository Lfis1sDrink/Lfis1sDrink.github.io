Markdown Content:
我们在做KYC的时候都会有活体认证这一步，也就是要对比手机前的人脸 VS 证件照的人脸，来保证是证件的本人在进行操作。

第一次攻防：欺诈者露脸操作

场景：使用他人的证件 + 欺诈者本人操作。

特征：手机捕捉的人脸特征 不匹配 证件照的人脸特征。

解决方案：人脸比对

欺诈者如何攻破：使用他人的证件 + 使用他人的人脸照片

第二次攻防：欺诈者使用静态照片

场景：使用他人的证件 + 摄像头前使用静态照片。

特征：手机捕捉的人脸特征 匹配 证件照的人脸特征，但是手机捕捉的人脸不会动。

解决方案：活体识别（张嘴 眨眼）

欺诈者如何攻破：使用提前录制好的活体视频（或者使用换脸的视频）

第三次攻防：欺诈者使用动态视频

场景：使用他人的证件 + 摄像头前使用准备好的视频。

特征：手机捕捉的人脸特征 匹配 证件照的人脸特征，且手机捕捉的人脸会动，但是点头、摇头、眨眼的动作和app要求的不一致。

解决方案：活体识别的动作加入随机顺序

欺诈者如何攻破：使用提前录制好的活体视频（或者使用换脸的视频），且不断尝试（一般活体失败后都允许重新尝试）

第四次攻防：欺诈者多次使用动态视频

场景：使用他人的证件 + 摄像头前使用准备好的视频并多次尝试（因为动作的组合比较少，总能撞上）。

特征：手机捕捉的人脸特征 匹配 证件照的人脸特征，且手机捕捉的人脸会动，点头、摇头、眨眼的动作和app要求的一致。

解决方案：加入不同的光，光照在人脸上是漫反射，照在（电脑/ipad/手机）屏幕上是镜面反射，以此判断摄像头前是真人还是照片。

欺诈者如何攻破：使用提前录制好的活体视频（或者使用换脸的视频），并且挡住手机屏幕（挡住光，防止光反射）。

第五次攻防：欺诈者多次使用动态视频+挡住手机光照

场景：使用他人的证件 + 摄像头前使用准备好的视频并多次尝试（因为动作的组合比较少，总能撞上） + 挡住手机发出的光照。

特征：手机捕捉的人脸特征 匹配 证件照的人脸特征，且手机捕捉的人脸会动，点头、摇头、眨眼的动作和app要求的一致，且人脸的光线为漫反射（没识别为异常）。

解决方案：引入不同的光颜色，并且顺序随机，对比光颜色变化顺序。

想深入了解的可以去看看腾讯的论文《Aurora Guard: Real-Time Face Anti-Spoofing via Light Reflection》。
